Complete Project File Guide

This page explains every file in this project so a teammate can quickly understand what each file does, what content it contains, and where it is used.

Project Structure Summary

  • Root HTML pages: 6 files
  • Shared includes: 2 files in inc/
  • JavaScript files: 3 files in js/
  • CSS files: 2 files in css/
  • Images: 9 files in img/
  • Total files: 22 tracked project files

Root HTML Files

index.html

Home page. Contains hero section, heritage section, collection preview cards, and CTA links to collections and inquiries. Uses local and external images plus dynamic greeting text.

.html

Collections page. Contains product grid cards with category metadata (data-category), filter buttons (All/Men/Women), and pricing. Works with filterItems() from js/custom.js.

board.html

Board page. Displays trustees/members in two repeated card grids with hover overlays and role descriptions. Uses four local member photos.

events.html

Events page. Contains a showcase hero with countdown placeholders and a list of upcoming events (Lagos, Abuja, Enugu), each with action buttons.

inquiries.html

Inquiries page. Includes booking form fields (name, email, service, date, message), validation message placeholders, and a hidden success state shown after valid submission.

dom-structure.html

Documentation page that explains shared website DOM architecture and reusable components (header, footer, JS layer, styling layer).

Shared Includes (inc/ )

inc/header.html

Global top navigation component. Includes links to all major pages, theme toggle button, and utility icons (search, shopping bag, mobile menu).

inc/footer.html

Global footer component. Contains animated marquee, brand summary, explore links, contact block, and bottom legal links. This file is injected into every page.

JavaScript Files (js/)

js/head.js

Injects shared CSS links (css/fonts.css and css/style.css) into <head> after DOM load.

js/tailwind-config.js

Defines Tailwind custom theme values: primary color, light/dark backgrounds, display font, and border-radius scale. Enables class-based dark mode.

js/custom.js

Main behavior file. Loads header/footer includes, handles product filtering, sets time-of-day greeting, validates inquiry form and success state, and controls theme toggle with localStorage persistence.

CSS Files (css/)

css/fonts.css

Imports Google fonts (Material Symbols and Manrope), sets base body font, and configures Material Symbols rendering properties.

css/style.css

Defines global light/dark body fallback colors, brand-color helper, marquee keyframes/animation utility, and active filter button style.
